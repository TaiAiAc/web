# 要做什么

  - 架构使用工作空间
  - 所有选项皆可覆盖 向外暴露
  - 只提供default选项

## 做一个内置vite vite各种插件的构建工具

  - 命令行
  - 专属配置文件
  - 要有各种插件内置
    - 远程静态资源构建时下载并替换引入路径
  - 针对构建输出定制

## 统一unocss配置
  - 集成unocss插件
  - 相关命令行工具
  - ui方面icon类插件
  - 定制主题 class类 变量类 函数类 等等

## ui组件库 naiveui相关
  - 表格表单详情页等等
  - 还要有业务支持的组件 业务组件 提供各种定制化插槽 自定义拓展 
  - 时间选择器封装 v-model:start="xxx" v-model:end="xxx" v-model:value="xxx"
  - 上传组件 支持多文件上传 自定义上传逻辑 自定义上传参数
  - 表单组件就算不在form里也支持独立校验
  - 预置一些表单通用的校验方式 开放接口
  - 考虑写一个校验中台服务提供异步校验
  - 组件有全局配置 在app.use的时候注入全局的组件配置

## 可视化图表类的库
  - Echarts、antV 之类的
  - 封装成vue组件
  - 并提供 use 方法

## http框架 支持多url配置 建立类型系统 
  - 考虑接入node做网关 统一接口返回类型 
  - 框架必须有详尽的ts类型系统
  - 还要有异步接口组件 提供各种定制化插槽 自定义拓展 

## js ts 工具类
  - 提供一些常用的js ts 工具类 比如 日期格式化 数组操作 字符串操作 等等

## 项目规范类工具
  - git 提交规范
  - eslint 规范
  - 统一使用 eslint 格式化代码
  - 项目有changelog 记录每次更新

## 开发类规范
  - pr 规范
  - 代码审查

## 权限模块
  - 提供权限相关的组件 比如 权限判断 权限路由 等等
  - 设计一套可抽离的权限判断系统
  - 一些辅助权限判断工具

    + 兼容旧版本配置项
